## 任务要求

- 实体间有哪些关系类型？（causes, located_in, treated_by, confirmed_by, associated_with, grading 等）

### 2. 实体抽取策略

  - 比较两者的优劣、成本和准确性
  - 建议的组合策略（hybrid approach）
- 给出抽取 pipeline 的伪代码

### 3. 图谱 Schema 设计
- 节点类型（label）设计

### 4. 数据质量控制

- 重复实体如何合并（dedup）？
- 实体对齐（同义词合并）策略
- 人工审核流程设计（最少人工介入原则）

### 5. 增量迭代策略
- 第一版 KG 的最小可行规模是多少节点/边？
- 新数据如何增量导入而不会破坏已有推理？
- 版本标记机制（每个节点带 version 字段）

### 6. 与推理管线的对接
- orchestrator 如何从 KG 中检索诊断路径？
- diagnostic_matcher 如何利用 KG 中的规则数据？
- 数据格式失配时的适配层设计

### 7. 验证方案
- 如何验证 KG 的完整性？（所有症状都能找到对应的诊断路径？）
- 如何验证 KG 的一致性？（没有孤立节点、没有矛盾关系？）
- 自动化验证脚本设计
