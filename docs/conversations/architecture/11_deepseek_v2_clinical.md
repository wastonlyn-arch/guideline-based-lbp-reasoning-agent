# DeepSeek 抽取策略建议（v2 — 临床PT/医生视角）

> **来源**：DeepSeek（临床物理治疗师/康复医生视角）
> **日期**：2026-05-11
> **核心观点**：L0-L8 + 子层，明确区分病理机制和症状，拆分干预逻辑，纳关键解剖变异

---

## 一、逐项决策

### 🔴 决策1：层结构
**✅ B) 采纳 L0-L8 + 子层**

原有的L0-L6更像"事物分类目录"，而不是"临床决策引擎"。病理生理机制(L3)与症状(L4)必须分开——椎间盘突出是病理实体，腰痛是主观体验。干预(L5)必须拆分，急性期患者，系统必须优先推荐祛因策略，而非训练。

### 🔴 决策2：L1 细化
**✅ B) 按损伤路径细分**

"错误的力学模式导致损伤"是Mcgill核心。Flexion+Compression → Delaminates Annulus → Disc Herniation 是一条完全不同于 Extension+Load → Pars Interarticularis → Spondylolysis 的路径。

### 🔴 决策3：诊断逻辑层
**✅ A) 不加（但作为MVP后的首要升级）**

这是一个优先级判断。可以通过更简洁的方式实现部分诊断能力——将诊断逻辑作为特定L4（体征/检查）节点的属性，而不必引入独立一层。

### 🔴 决策4：干预拆分
**✅ B) 拆为祛因 + 训练**

毫无妥协余地。系统必须有能力区分 Removing_the_Cause (L5a) 和 Building_Capacity (L5b)。

### 🟡 决策5：L3/L4 拆分
**✅ B) 拆为 机制 vs 症状**

Radiculopathy (L3: 神经病理) 和 Sciatica (L4: 患者说的"坐骨神经痛") 不是同义词。不拆分会导致系统犯常识性错误。

### 🟡 决策6：Anatomical Variation
**✅ B) 在L2/L3层侧预留并加入关键节点**

Mcgill反复强调"因人而异"。建议在MVP阶段将少数关键解剖变异概念化，作为 Anatomical_Factor 加入 L2 或 L3 层。

---

## 二、最终决策总结

**升级架构**：从7层扩展到8层，明确区分病理机制和症状，并拆分干预逻辑。
**深化内容**：细化L1生物力学和纳入关键解剖变异，提升"专业解释力"。
**控制复杂度**：将独立的诊断逻辑层延后，通过强化现有节点属性，在MVP阶段实现核心诊断功能。
