# 可计算康复医学：分层医疗认知 Agent（MVP）设计文档

## 1. 项目背景与哲学

### 1.1 为什么需要分层医疗 Agent？

- 通用大模型（如 ChatGPT）缺乏可解释的因果链，无法满足医疗领域的可审计要求。
- 医学知识本质是**跨层级的概率映射**（分子→细胞→组织→个体→群体），且存在大量中间机制（如生物力学、炎症反应）。
- 临床决策需要**既知道"什么"也知道"为什么"**，并能够追溯到原始证据。

### 1.2 项目哲学

> 现象是冰山浮在水面的一角，本质是隐藏在水下的巨大结构。医疗 Agent 的任务就是显式地构建从现象到本质的映射路径。

- 每一个临床禁忌（如"腰痛避免弯腰"）背后都隐藏着一串因果链（弯腰→剪切力增加→椎间盘负荷→神经受压→疼痛）。
- 通过将中间概念显式化为节点，我们可以实现**可解释推理**，并支持干预不同层级（例如减少剪切力，而不是单纯避免动作）。

## 2. 核心架构：分层知识网络 + 混合检索

```
┌─────────────────────────────────────────────────────────────────┐
│ 输入层（患者信息）                                               │
│ 主诉、体征、影像检查、既往病史                                   │
└─────────────────────────────┬───────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 实体抽取（GLiNER / 规则）                                        │
│ 提取症状、动作、解剖位置等概念                                   │
└─────────────────────────────┬───────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 混合检索引擎                                                     │
│ ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐    │
│ │ 图谱路径查询     │ │ 向量检索（RAG）  │ │ 证据排序         │    │
│ │ (SQLite递归CTE)  │ │ (FAISS)         │ │ (confidence)    │    │
│ └─────────────────┘ └─────────────────┘ └─────────────────┘    │
└─────────────────────────────┬───────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 提示词构建 + 大模型生成                                          │
│ 将路径、文献片段、患者信息组装为富上下文 → LLM（DeepSeek/Ollama） │
└─────────────────────────────┬───────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 输出层（SOAP格式报告）                                           │
│ S（主观） / O（客观） / A（评估） / P（计划）                     │
│ 每条结论附证据来源（图谱路径 / 文献引用）                         │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 知识分层模型（以下腰痛为例）

| 层级 | 概念示例 | 作用 |
|------|----------|------|
| L0 动作/行为 | 弯腰、提重物、坐位前倾 | 诱因 |
| L1 生物力学 | 椎间盘剪切力、椎间盘内压、核心稳定性 | 力学机制 |
| L2 病理/组织 | 纤维环破裂、髓核突出、炎症因子 | 结构性改变 |
| L3 神经/症状 | 神经根受压、放射痛、局部疼痛 | 临床表现 |
| L4 体征/检查 | 直腿抬高试验阳性、MRI 突出分级 | 诊断证据 |
| L5 干预/康复 | 核心稳定训练、神经动态松动、避免屈曲 | 治疗措施 |
| L6 预后/结局 | 疼痛缓解、重返工作、功能恢复 | 康复目标 |

**分层的好处**：
- 推理时限定范围（例如只查询生物力学层）。
- 干预可以作用于任意层（如通过训练核心肌群减少剪切力，而不是单纯禁止弯腰）。
- 支持"为什么"追问（沿着路径向上/下游追溯）。

## 4. 知识图谱中的关系类型

| 类别 | 关系 | 示例 | 用途 |
|------|------|------|------|
| 层级/组成 | `is_a` | "腰椎间盘突出" is_a "椎间盘退行性疾病" | 泛化推理 |
| | `part_of` | "髓核" part_of "椎间盘" | 解剖组成 |
| 临床关联 | `associated_with` | "吸烟" associated_with "椎间盘退变" | 风险因素 |
| | `predisposes_to` | "肥胖" predisposes_to "腰椎间盘突出" | 易感性 |
| | `differential_with` | "腰椎间盘突出" differential_with "骶髂关节紊乱" | 鉴别诊断 |
| 时空/解剖 | `located_in` | "突出物" located_in "L4/L5" | 定位 |
| | `precedes` | "纤维环破裂" precedes "髓核突出" | 时间顺序 |
| 干预/结局 | `treats` | "核心稳定训练" treats "非特异性下腰痛" | 治疗 |
| | `contraindicated_for` | "弯腰" contraindicated_for "急性椎间盘突出" | 禁忌 |
| | `indicated_for` | "手术" indicated_for "进行性神经功能缺损" | 适应证 |
| 比较/证据 | `more_effective_than` | "麦肯基疗法" more_effective_than "超声" 对于急性腰痛 | 指南推荐 |
| | `similar_to` | "动态神经松动" similar_to "静态拉伸" | 机制类比 |
| 元知识 | `derived_from` | "禁忌" derived_from "NICE指南2023" | 来源追溯 |

**MVP 优先实现**：`is_a`, `part_of`, `treats`, `contraindicated_for`, `associated_with`。

## 5. 技术选型（MVP）

| 组件 | 选型 | 理由 |
|------|------|------|
| **知识图谱存储** | SQLite + 递归 CTE | 零部署，单文件，适合 ≤2000 节点 |
| **向量检索** | FAISS (CPU) + sentence-transformers (all-MiniLM-L6-v2) | 轻量，无需 GPU |
| **实体抽取** | GLiNER（零样本 NER）或 spaCy 规则 | 本地运行，不依赖 API |
| **图谱路径查询** | SQLite 递归 CTE，最大深度 3 | 简单可控 |
| **大模型推理** | DeepSeek API（成本极低）或 Ollama（完全离线） | 推荐 DeepSeek（1M token < 0.5元） |
| **编排/工作流** | Python 函数 + 条件分支 | MVP 阶段避免过度工程 |
| **演示环境** | Jupyter Notebook | 快速展示推理过程 |

## 6. 工程实现步骤

### 6.1 数据准备（手工 + 半自动）

1. **构建本体**：手工录入 20-30 个核心概念（如腰椎间盘突出、放射痛、核心稳定训练），并标注其层级。
2. **构建关系**：手工录入 30-50 条边（`causes`, `treats`, `contraindicated_for` 等），每条边可带 `confidence` 分数。
3. **收集文献片段**：从教材或指南复制 5-10 段与上述概念相关的原文，存入 `chunks` 表，并关联到对应节点。

### 6.2 数据库表结构（SQLite）

```sql
CREATE TABLE nodes (
    id INTEGER PRIMARY KEY,
    name TEXT UNIQUE,
    layer TEXT,          -- 如 biomechanics, pathology, symptom, treatment
    description TEXT
);

CREATE TABLE edges (
    id INTEGER PRIMARY KEY,
    source_id INTEGER,
    target_id INTEGER,
    relation TEXT,       -- 如 causes, treats, contraindicated_for
    confidence REAL,     -- 0-1
    source_ref TEXT,     -- 文献或指南标识
    FOREIGN KEY(source_id) REFERENCES nodes(id),
    FOREIGN KEY(target_id) REFERENCES nodes(id)
);

CREATE TABLE chunks (
    id INTEGER PRIMARY KEY,
    text TEXT,
    source TEXT,         -- 文献标题或教材章节
    node_id INTEGER,     -- 可选，关联到相关概念
    FOREIGN KEY(node_id) REFERENCES nodes(id)
);
```

### 6.3 核心函数

| 函数 | 职责 |
|------|------|
| `extract_entities(text)` | 调用 GLiNER 提取症状、动作、解剖位置 |
| `retrieve_graph_paths(entity, max_depth=3)` | 递归 CTE 返回因果/关联路径 |
| `retrieve_vector_chunks(query, top_k=3)` | FAISS 检索相关文献段落 |
| `build_enhanced_prompt(patient_info, paths, chunks)` | 组装成最终提示词 |
| `call_llm(prompt)` | 调用 DeepSeek API 或 Ollama |

### 6.4 主流程

```python
def medical_agent(patient_text):
    entities = extract_entities(patient_text)
    paths = []
    for ent in entities:
        paths.extend(retrieve_graph_paths(ent, max_depth=3))
    chunks = retrieve_vector_chunks(patient_text, top_k=3)
    prompt = build_enhanced_prompt(patient_text, paths, chunks)
    report = call_llm(prompt)
    return report
```
