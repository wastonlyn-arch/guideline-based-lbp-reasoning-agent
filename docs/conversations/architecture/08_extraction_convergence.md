## 2. 逐议题展开

### 2.1 🟢 L1 需要细化（共识）

| 方 | 具体建议 |
|:--:|:---------|
| 你 | 拆为 L1（负荷）+ L1.5（负荷模式） |
| ChatGPT | 区分"做什么"(L0)和"产生什么"(L1) |
| DeepSeek | L1 按损伤路径细化：Flexion+compression / Shear / Repeated low load |

**要点**：大家对"L1太粗"一致，但具体怎么细有分歧。

### 2.2 🟡 是否需要独立的"诊断逻辑层"

| 方 | 立场 |
|:--:|:------|
| 你 | 未涉及 |
| ChatGPT | 未明确提出 |
| DeepSeek | **强烈建议新增 L4.5/L6 Diagnostic_logic**，并称"否则只是知识库，不是推理系统" |

**关键分歧**：这是 DeepSeek 最核心的建议，但其他方未直接回应。

### 2.3 🟢 干预层需要拆分（共识）

| 方 | 具体方案 |
|:--:|:---------|
| ChatGPT | L5a 祛因（避免动作、改变模式）+ L5b 训练（核心稳定、手术） |
| DeepSeek | L7 干预，含 Remove cause + Exercise + Medication + Surgery |

**要点**：大家都认为"去除诱因"和"训练"是两件事，应在图谱中体现。

### 2.4 🟡 L3/L4 边界

| 方 | 立场 |
|:--:|:------|
| 你 | L3=神经/症状，L3.5=症状 |
| ChatGPT | 未明确拆分 |
| DeepSeek | **必须拆**为 L3=Pathophysiology（机制）, L4=Symptoms（体验） |

### 2.5 🔴 ChatGPT 独家：Anatomical Variation

ChatGPT 提出增加 `Anatomical_Variation` 维度（如 `Deep_Hip_Socket`, `Oval_Disc_Shape`），解释同一动作对不同人的差异。DeepSeek 未提及。

## 3. 与 03_architecture.md 的冲突

你的 **03_architecture.md** 已经定义了 L0-L6 的固定结构。本轮讨论提出的修改如果全部采纳，会变成 L0-L8 + 子层，与已有架构文档不一致。

**需要你决策的冲突点**：

| 冲突 | 03_architecture.md | 本轮建议 | 影响 |
|:----:|:------------------:|:--------:|:----:|
| 层数 | 7层 (L0-L6) | 8-9层 | schema.sql 需修改，已有 init_data 格式要变 |
| L1 | 单层 biomechanics | 细化为损伤路径 | 节点命名和 type 要扩展 |
| L3/L4 | L3=神经/症状, L4=体征 | 拆为 L3=机制, L4=症状, L5=检查 | 影响 graph_query 的路径约束逻辑 |
| 干预 | L5=干预 | 拆为祛因+训练 | 路径查询需新增 path_type |
| 诊断逻辑 | 无 | DeepSeek 建议新增 | 新模块，影响最大 |
