# 康复知识图谱抽取策略（Reference / 工具书版）

> **来源**：项目所有者
> **日期**：2026-05-11
> **用途**：本轮抽取策略讨论的原始提案，供三家 AI 评估

---

## 一、总体目标

构建一个结构化、模块化、可检索的知识库，满足：
- 快速查找（类似词典 / 手册）
- 机制解释（但不展开过度推理）
- 支持后续：教学路径（View 1）、临床决策（View 3）

## 二、核心设计原则

1. **"概念优先"，不是"路径优先"** — 工具书的核心不是完整推理链，而是每个概念都能被独立理解 + 可被引用
2. **每个节点必须满足"三要素"** — 定义、机制、关联
3. **控制复杂度** — 不强制所有链条闭环，允许"弱关系存在"

## 三、实体抽取规范

**命名**：PascalCase + 下划线，如 `Lumbar_Flexion`, `Disc_Compression`

**必选字段**：
```json
{"name": "Lumbar_Flexion", "layer": "L0", "type": "action", "description": "腰椎屈曲动作"}
```

**类型分类**：action / load / exposure / pathology / mechanism / symptom / sign / test / intervention / outcome

**分层映射**：

| 层级 | 内容 |
|:----:|:----:|
| L0 | 动作 |
| L1 | 负荷 |
| L1.5 | 负荷模式 |
| L2 | 病理 |
| L3 | 神经 |
| L3.5 | 症状 |
| L4 | 检查 |
| L5 | 干预 |
| L6 | 预后 |

**否定信息**：必须保留，`negated: true`

## 四、关系抽取规范

**允许关系**：causes / increases / decreases / associated_with / indicated_for / contraindicated_for / precedes / treated_by

**方向固定**：动作→负荷→病理→神经→症状→检查→干预

## 五、证据规范

- 20-50字，来自原文
- 置信度：0.9=明确机制 0.7=明确陈述 0.5=相关性 <0.4不建议纳入

## 六、抽取优先级

| 优先级 | 内容 |
|:------:|:----:|
| 🥇 一级 | 动作→负荷→病理→症状（基础机制库） |
| 🥈 二级 | 检查→指示，干预→作用 |
| 🥉 三级 | 影像→病理（弱关系），非特异性描述 |

## 七、错误知识（必须显式保留）

工具书必须包含"哪些常见做法是错误的"。

## 八、定位总结

> 这是一个"结构化医学词典 + 机制索引系统"
