## 2. 逐项决策对比

### 2.1 层数（共识：都需要扩展）

| 方 | 具体方案 | 层数 |
|:--:|:---------|:----:|
| ChatGPT v3 临床 | L0-L8，含诊断逻辑层 | 9层 |
| ChatGPT v4 保守 | L0-L7，不加诊断层，用type细化 | 8层 |
| DeepSeek v2 临床 | L0-L8，干预拆子类 | 8层+子层 |

**共识**：🟢 都认为L0-L6不够，需要扩展到8层左右
**分歧**：🟡 具体8还是9层，诊断层加不加

### 2.2 L1 细化（完全共识）

| 方 | 立场 |
|:--:|:------|
| ChatGPT v3 | ⭐ 必须细化，是McGill体系核心 |
| ChatGPT v4 | ✅ 必须细化，但用type体系，不增加物理层 |
| DeepSeek v2 | ✅ 必须按损伤路径细分 |

**共识**：🟢 全票通过L1细化

### 2.3 ⚠️ 诊断逻辑层（最大分歧）

| 方 | 立场 | 理由 |
|:--:|:----:|:------|
| ChatGPT v3 | ✅ **必须加**，否则"不是推理系统" | 模式识别是临床核心，不加就是医学百科 |
| ChatGPT v4 | ❌ **明确反对** | 诊断是推理行为不是知识节点，用"边+路径"表达 |
| DeepSeek v2 | ⏳ **MVP不加，但作为首要升级** | 可先将诊断逻辑作为节点属性融入L4 |

**共识**：🟡 ChatGPT v4和DeepSeek v2都认为MVP阶段可以不加（但理由不同）
**分歧**：🔴 ChatGPT v3认为必须现在加

### 2.4 干预拆分（完全共识）

| 方 | 立场 |
|:--:|:------|
| ChatGPT v3 | ✅ 拆为 L7a祛因 / L7b训练 / L7c医疗 |
| ChatGPT v4 | ✅ 拆为 Remove_cause / Exercise / Passive / Medication / Surgery |
| DeepSeek v2 | ✅ 拆为 Removing_the_Cause(L5a) + Building_Capacity(L5b) |

**共识**：🟢 全票通过，毫无争议

### 2.5 L3/L4 拆分（完全共识）

| 方 | 立场 |
|:--:|:------|
| ChatGPT v3 | ✅ 必须拆（L3病理机制 / L4症状 / L5检查） |
| ChatGPT v4 | ✅ 必须拆（L3神经机制 / L4症状 / L5体征） |
| DeepSeek v2 | ✅ 必须拆（L3病理机制 / L4症状） |

**共识**：🟢 全票通过

### 2.6 Anatomical Variation

| 方 | 立场 |
|:--:|:------|
| ChatGPT v3 | ⏳ MVP不加，第二阶段再做 |
| ChatGPT v4 | ✅ 保留但不作为层，作为节点属性 |
| DeepSeek v2 | ✅ MVP阶段在L2/L3层加入关键节点 |

**共识**：🟡 都认为有必要，但"当不当作层"有分歧

### 2.7 错误知识保留

| 方 | 立场 |
|:--:|:------|
| ChatGPT v3 | ✅ 必须保留 |
| ChatGPT v4 | ✅ 强烈建议提升优先级 |
| DeepSeek v2 | 未直接提及 |

**共识**：🟢 你和前两者一致
